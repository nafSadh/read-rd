Section 01

What is a Vector?

A vector is not just an arrow. Depending on who you ask, it might be a geometric arrow with a direction and magnitude, a column of numbers in a tuple, a polynomial, an audio signal, or a function. The abstract algebraic view unifies all of these: a vector is any object that lives inside a vector space—a set where you can add elements together and scale them by numbers, and those operations behave sensibly.

In machine learning, vectors are the lingua franca of data. Each feature becomes a dimension, each data point becomes a vector in Rn, and the entire dataset becomes a collection of points in high-dimensional space. 3Blue1Brown puts it well: the physics student, the CS student, and the math student each see a different vector, but the algebraic rules they share are what matter.

This section sets the stage for everything that follows. Understanding what a vector is—abstractly, not just as a tuple—is what lets you later see why polynomials, matrices, and even functions can be treated with the same linear algebra toolkit.

Section 02

Matrices & Operations

A matrix is a rectangular array of numbers, but thinking of it as just a table misses the point. Matrices serve two roles simultaneously: they are compact notation for systems of linear equations and they represent linear mappings between vector spaces. Every time you multiply a matrix by a vector, you are applying a transformation.

Matrix addition and scalar multiplication work element-wise. Matrix multiplication is more interesting: it composes transformations (apply A first, then B, so you write BA). Crucially, matrix multiplication is not commutative—AB and BA are generally different, and sometimes only one of them is even defined. The transpose flips rows and columns; the inverse undoes a transformation, but only exists when the matrix is square and full rank.

Solving Linear Systems

The question Ax = b is everywhere in ML: fit a regression, solve for weights, invert a transformation. Gaussian elimination transforms the augmented matrix [A|b] into row echelon form (REF) through elementary row operations—swap, scale, and subtract—then reads off solutions by back-substitution. Reducing further to RREF makes the solution even more explicit.

The number of pivots equals the rank and determines everything: unique solution, infinitely many, or none at all. Free variables correspond to directions you can move in without changing Ax. The general solution is a particular solution plus any element of the null space. MML's "minus-1 trick" is an elegant shortcut for reading the null space directly from RREF by inserting -1 rows at non-pivot positions.

Section 04

Vector Spaces & Subspaces

Before vectors can be added and scaled, there must be a structure that guarantees those operations make sense. A vector space (V, +, ·) is a set equipped with addition and scalar multiplication that satisfies eight axioms (closure, associativity, commutativity, identities, inverses, distributivity). The construction builds on groups: (V, +) must be an abelian group, and scalar multiplication must distribute over both vector addition and field addition.

A subspace is a subset of V that is itself a vector space under the same operations. The test is simple: closed under addition and scalar multiplication, and contains the zero vector. The solution set of Ax = 0 always passes this test (it is the null space, a subspace). But the solution set of Ax = b for b ≠ 0 fails: it does not contain the zero vector, so it is an affine subspace—a shifted copy of the null space.

Section 05

Linear Independence

A set of vectors is linearly independent if no vector in the set can be written as a linear combination of the others. Equivalently, the only way to get the zero vector from a linear combination of them is to set all coefficients to zero. If you can write one vector in terms of the others, the set is linearly dependent—there is redundancy.

Think of flying from Nairobi to Kampala to Kigali: if the Kigali direction is just a combination of the Nairobi and Kampala directions, you have not gained a genuinely new direction. Testing independence boils down to Gaussian elimination: stack the vectors as columns, row-reduce, and count pivots. Fewer pivots than columns means dependence.

Section 06

Basis, Rank & Dimension

A generating set (or spanning set) is a collection of vectors whose linear combinations fill up a vector space. A basis is a generating set that is also linearly independent—it has no redundancy. Every basis of a given vector space has the same number of elements; that number is the dimension.

The rank of a matrix is the dimension of the column space (equivalently, the row space). It equals the number of pivots after Gaussian elimination. A basis is simultaneously the minimal generating set and the maximal linearly independent set. These are two sides of the same coin, and understanding that duality is one of the most clarifying insights in all of linear algebra.

Section 07

Linear Mappings

A linear mapping (or linear transformation) is a function between vector spaces that preserves the two fundamental operations: Φ(x + y) = Φ(x) + Φ(y) and Φ(λx) = λΦ(x). This preservation of structure is the defining idea. Rotations, reflections, scaling, shearing, projections—all linear maps. Translation is not linear (it does not send the zero vector to zero).

The deep theorem: once you fix bases for the source and target vector spaces, every linear mapping corresponds to exactly one matrix, and every matrix corresponds to exactly one linear mapping. This means studying matrices is studying linear maps. A mapping can be injective (one-to-one), surjective (onto), or bijective (both). Bijective linear maps are isomorphisms—the two spaces are structurally identical.

Injective

One-to-one: distinct inputs produce distinct outputs. Kernel = {0}.

Surjective

Onto: every element in the target space is reached. Image = codomain.

Bijective

Both injective and surjective. The map has an inverse. Isomorphism.

Transformation Matrix

Fix bases, read off columns: the j-th column is Φ(b_j) expressed in target basis.

Section 08

Image, Kernel & Connections

Every linear map Φ has two fundamental subspaces attached to it. The image (Im Φ) is the set of all outputs—everything the map can reach—which for a matrix A is its column space. The kernel (ker Φ) is the set of all inputs that map to zero—the null space. The rank-nullity theorem ties them together: dim(Im) + dim(ker) = dim(domain).

This decomposition is the backbone of applied linear algebra. Strang's "four fundamental subspaces" picture connects column space, null space, row space, and left null space into a single coherent framework. These ideas recur throughout the MML book: in Chapter 4 (determinants), Chapter 9 (linear regression), Chapter 10 (dimensionality reduction), and Chapter 12 (classification). Change of basis shows how the same linear map looks different in different coordinate systems—the matrix changes, but the map does not.

Section 01 — Details

What is a Vector?

The word "vector" hides a surprising amount of depth. In a physics classroom, a vector is an arrow with magnitude and direction. In a computer science course, it is an ordered list of numbers—a tuple in Rn. In abstract algebra, a vector is any element of a vector space, which is a set V equipped with two operations (addition and scalar multiplication) satisfying eight axioms. The MML book takes the abstract path from the start, and for good reason: if you only think of vectors as arrows or tuples, you will be confused the first time someone hands you a polynomial or a function and calls it a vector.

The eight axioms can be grouped into two clusters. First, (V, +) must be an abelian group: addition is associative, commutative, has an identity (the zero vector), and every element has an additive inverse. Second, scalar multiplication must be compatible: it distributes over both vector addition and scalar addition, it is associative with field multiplication, and 1 · x = x. Any set satisfying these axioms is a vector space, regardless of what the "vectors" look like. Polynomials of degree at most n form a vector space. Continuous functions on [0, 1] form a vector space. Even m × n matrices form a vector space.

Three perspectives on vectors

3Blue1Brown's Essence of Linear Algebra opens by contrasting three viewpoints. The physics student sees geometric arrows rooted at the origin. The CS student sees ordered lists of numbers—coordinates that index features. The mathematics student sees abstract elements of a set that obeys certain rules. All three viewpoints are correct and useful. The geometric viewpoint gives you intuition about direction, length, and angle. The coordinate viewpoint lets you compute. The abstract viewpoint lets you generalize to spaces that have no obvious geometry, like the space of all solutions to a differential equation.

In machine learning, the coordinate viewpoint dominates day-to-day work: each data point is a vector in Rd where d is the number of features. But the abstract viewpoint is what lets you understand why the same algorithms work across wildly different domains. Word embeddings, pixel arrays, audio spectrograms, graph node features—they all live in vector spaces, so the same linear algebra tools apply. The map from raw data to a feature vector is itself a modeling choice, and understanding what a vector space requires helps you evaluate whether a given representation is appropriate.

Beyond arrows: exotic vector spaces

Consider the set of all polynomials of degree at most 2: p(x) = a + bx + cx2. You can add two such polynomials and get another polynomial of degree at most 2. You can scale one by a real number and stay in the set. The zero polynomial serves as the zero vector. All eight axioms hold, so this set is a vector space of dimension 3 (a natural basis is {1, x, x2}). This example matters because it shows that vectors do not have to be columns of numbers. In kernel methods, you often work implicitly in infinite-dimensional vector spaces of functions—and the linear algebra still works.

Key Insight

The power of the abstract definition is generality: once you verify the eight axioms, every theorem in linear algebra applies automatically. You do not need to re-prove anything for polynomials or functions or matrices—they inherit the entire theory for free.

Section 02 — Details

Matrices & Operations

A matrix A in Rm×n has m rows and n columns. The entry in row i, column j is aij. But the real way to think about a matrix is as a linear mapping from Rn to Rm: the function f(x) = Ax takes an n-dimensional input vector and produces an m-dimensional output vector. When you multiply A by x, you are forming a linear combination of the columns of A, weighted by the entries of x. Column j of A tells you where the j-th standard basis vector ej lands under the transformation.

Matrix addition is straightforward: (A + B)ij = aij + bij, requiring both matrices to be the same size. Scalar multiplication scales every entry. Matrix multiplication is the operation that encodes composition: if A maps Rn to Rm and B maps Rm to Rp, then BA maps Rn to Rp. The (i, j) entry of BA is the dot product of row i of B with column j of A. This is why the inner dimensions must match: (p × m)(m × n) = (p × n).

Why multiplication is not commutative

Commutativity fails for matrix multiplication because order of transformation matters. Rotating a shape and then shearing it gives a different result from shearing first and then rotating. Concretely, if A is 2×3 and B is 3×2, then AB is 2×2 and BA is 3×3—they are not even the same size. Even for square matrices of the same size, AB and BA are generally different. This non-commutativity is not a defect; it reflects the physical reality that the order of operations matters.

Inverse and transpose

A square matrix A in Rn×n is invertible if there exists a matrix A-1 such that AA-1 = A-1A = I. Invertibility is equivalent to the matrix having full rank (rank n), having a non-zero determinant, having no zero eigenvalues, and the linear system Ax = b having a unique solution for every b. When the matrix is not invertible (singular), some information is lost in the transformation, and you cannot recover the input from the output.

The transpose AT swaps rows and columns: (AT)ij = aji. Useful identities: (AB)T = BTAT (order reverses), (AT)T = A, and (A + B)T = AT + BT. Symmetric matrices satisfy A = AT and have special properties (real eigenvalues, orthogonal eigenvectors). Covariance matrices, Hessians, and kernel matrices are all symmetric—you will encounter them constantly in ML.

Key Insight

Matrix multiplication is not commutative, but it is associative: (AB)C = A(BC). This is why neural network layers can be "grouped" in any order without changing the result—the composition is well-defined regardless of how you parenthesize it.

Section 03 — Details

Solving Linear Systems

A system of linear equations Ax = b, where A is m × n, asks: is there a vector x in Rn such that each row of A dotted with x equals the corresponding entry of b? Gaussian elimination answers this by performing elementary row operations on the augmented matrix [A | b]: swap two rows, multiply a row by a non-zero scalar, or add a multiple of one row to another. These operations preserve the solution set because each one is reversible.

The algorithm proceeds in two phases. Forward elimination creates upper-triangular form (row echelon form, REF) by introducing zeros below each pivot. Back-substitution then solves from the last equation upward. For a fully explicit solution, you can continue to reduced row echelon form (RREF), where each pivot is 1 and is the only non-zero entry in its column. RREF is unique for any given matrix—different sequences of row operations always arrive at the same RREF.

Pivots and the anatomy of solutions

A pivot is the leading non-zero entry in a row of the echelon form. The number of pivots equals the rank of A. Variables corresponding to pivot columns are called basic (or pivot) variables; the rest are free variables. If rank(A) = rank([A | b]), the system is consistent (at least one solution exists). If additionally rank(A) = n (number of unknowns), the solution is unique. If rank(A) < n, there are n - rank(A) free variables, and the solution set is an affine subspace of dimension n - rank(A).

The general solution takes the form x = xparticular + xnull, where xparticular is any single solution to Ax = b and xnull ranges over the entire null space of A. This decomposition is fundamental: the particular solution shifts you to the right "level" (satisfying Ax = b), and the null space tells you all the directions you can move without changing Ax.

The minus-1 trick

MML describes an elegant technique for reading the null space directly from RREF. After reducing A to RREF, identify the pivot and non-pivot (free) columns. Construct an extended matrix by inserting rows of the form (0, ..., 0, -1, 0, ..., 0) at positions corresponding to non-pivot columns. The columns of this extended matrix that correspond to non-pivot positions in the original matrix form a basis for the null space. This avoids the need for a separate back-substitution step to find the null space.

Key Insight

The general solution of Ax = b is always of the form x = x_particular + (any element of the null space). Solving a linear system therefore reduces to two sub-problems: finding one particular solution, and finding the null space of A.

Section 04 — Details

Vector Spaces & Subspaces

MML builds the definition of a vector space in layers. It begins with groups: a group (G, *) is a set with an operation that is associative, has an identity element, and has inverses for every element. If the operation is also commutative, it is an abelian group. A vector space (V, +, ·) then requires that (V, +) is an abelian group, and that scalar multiplication distributes and is compatible with the field operations. This layered construction might feel slow, but it makes the axioms easy to check: first verify the group properties of addition, then verify the scalar multiplication properties.

The standard example is Rn with component-wise addition and scalar multiplication. But there are many others. The set of all m × n matrices forms a vector space of dimension mn. The set of polynomials of degree at most d forms a vector space of dimension d + 1. The set of all continuous functions on [0, 1] forms an infinite-dimensional vector space. In each case, you can verify the axioms mechanically.

Subspaces: the closure test

A subspace U of V is a non-empty subset that is itself a vector space under the same operations. The shortcut: check (1) the zero vector is in U, (2) U is closed under addition, and (3) U is closed under scalar multiplication. If all three hold, U inherits every other axiom from V automatically. This is far easier than checking all eight axioms from scratch.

The null space of a matrix A (the set of all x with Ax = 0) is always a subspace of Rn. Proof: A · 0 = 0 (zero vector is in it); if Ax = 0 and Ay = 0, then A(x + y) = Ax + Ay = 0 (closure under addition); if Ax = 0, then A(λx) = λAx = 0 (closure under scalar multiplication). But the solution set of Ax = b for b ≠ 0 is not a subspace: it does not contain the zero vector (A · 0 = 0 ≠ b). Instead, it is an affine subspace—a translate of the null space by any particular solution xp.

Why the distinction matters

The subspace/affine subspace distinction recurs throughout ML. The set of all weight vectors achieving zero training loss on a linear model forms an affine subspace (if any solution exists). Regularization picks a specific point in that affine subspace, typically the one closest to the origin (for L2 regularization). Understanding this geometry helps you see what regularization actually does: it is not just "adding a penalty" but selecting among infinitely many solutions by projecting onto the nearest point in the null space direction.

Key Insight

A subspace must pass through the origin. The solution set of Ax = b (for b ≠ 0) is a copy of the null space, shifted away from the origin by one particular solution. This is why it is called an "affine" subspace: it has the shape of a subspace but not the location.

Section 05 — Details

Linear Independence

A set of vectors {v1, v2, ..., vk} in a vector space V is linearly independent if the equation λ1v1 + λ2v2 + ... + λkvk = 0 has only the trivial solution λ1 = λ2 = ... = λk = 0. If any non-trivial solution exists, the set is linearly dependent, which means at least one vector can be expressed as a linear combination of the others. Intuitively, a linearly independent set has no redundancy: removing any single vector shrinks the span.

To test independence, arrange the vectors as columns of a matrix and row-reduce. If every column contains a pivot, the set is independent. If any column lacks a pivot, the set is dependent—that column's vector can be written in terms of the pivot columns. This is a mechanical procedure: form the matrix, run Gaussian elimination, count pivots. In Rn, you can have at most n linearly independent vectors (the dimension of the space).

The Nairobi-Kampala-Kigali example

Imagine three cities in East Africa: Nairobi, Kampala, and Kigali. Suppose the displacement vector from a reference point to Nairobi is v1, to Kampala is v2, and to Kigali is v3. If Kigali happens to lie exactly on the line segment between Nairobi and Kampala (so v3 = αv1 + βv2 for some α, β), then knowing directions to Nairobi and Kampala already tells you how to get to Kigali. The third vector adds no new directional information. The set {v1, v2, v3} is linearly dependent. But if Kigali is off that line, then v3 provides a genuinely new direction, and the set is linearly independent.

Independence in practice

In machine learning, linear dependence among features means redundancy. If one feature is a perfect linear combination of others, the design matrix X does not have full column rank, and the normal equations XTXx = XTy have infinitely many solutions (the least-squares solution is not unique). This is called multicollinearity. Even approximate dependence causes numerical instability. PCA addresses this by projecting onto the most "independent" directions (eigenvectors of the covariance matrix), discarding dimensions with near-zero variance—those are the directions of near-dependence.

A related concept: a set of vectors is maximally linearly independent in V if adding any vector from V to the set makes it dependent. Such a set is exactly a basis of V. This connects independence directly to the concepts in the next section.

Key Insight

Linear independence is the "no redundancy" condition. A set of k vectors in Rn is independent if and only if the matrix formed by these vectors as columns has rank k. In Rn, you can never have more than n independent vectors.

Section 06 — Details

Basis, Rank & Dimension

A generating set (or spanning set) of a vector space V is a set of vectors A = {a1, ..., ak} such that every vector in V can be written as a linear combination of vectors in A. In other words, span(A) = V. A generating set might be larger than necessary—it can contain redundant vectors. A basis is a generating set that has been trimmed of all redundancy: it is both a generating set and linearly independent.

The remarkable fact is that every basis of a vector space V has the same number of elements. This invariant is called the dimension of V, written dim(V). In Rn, the dimension is n. The standard basis {e1, ..., en} (where ei has 1 in position i and 0 elsewhere) is the most familiar basis, but infinitely many other bases exist. Any n linearly independent vectors in Rn form a basis, as do any n vectors that span Rn.

The dual characterization

A basis can be characterized in two equivalent ways: (1) it is a minimal generating set—removing any element breaks the spanning property; (2) it is a maximal linearly independent set—adding any vector from V introduces dependence. These are two sides of the same coin. If you start with a generating set and remove redundant vectors one at a time, you eventually reach a basis. If you start with an independent set and add vectors one at a time (maintaining independence), you also reach a basis. Both processes terminate at the same size: dim(V).

Rank of a matrix

The rank of a matrix A, written rk(A), is the dimension of the column space (equivalently, the image of the linear mapping represented by A). It equals the number of linearly independent columns, which also equals the number of linearly independent rows, which also equals the number of pivots after Gaussian elimination. For A in Rm×n, rk(A) ≤ min(m, n). When rk(A) = min(m, n), the matrix is called full rank. Full rank is the generic case—a "random" matrix almost certainly has full rank.

Rank connects to solvability: rk(A) = n means the columns are independent and Ax = b has at most one solution for each b. rk(A) = m means the columns span Rm and Ax = b has at least one solution for every b. rk(A) = m = n means the matrix is invertible—exactly one solution for every b. In machine learning, the rank of a data matrix tells you the intrinsic dimensionality of the data. If 1000-dimensional data has rank 5, the data truly lives on a 5-dimensional subspace, and dimensionality reduction can recover it without loss.

Key Insight

Every basis of a given vector space has exactly the same number of elements. This number—the dimension—is an intrinsic property of the space, independent of which basis you choose. Rank is dimension applied to the column space of a matrix.

Section 07 — Details

Linear Mappings

A linear mapping (also called a linear transformation or linear map) is a function Φ: V → W between two vector spaces that preserves the two defining operations: Φ(x + y) = Φ(x) + Φ(y) for all x, y in V, and Φ(λx) = λΦ(x) for all x in V and scalars λ. Equivalently, Φ(λx + ψy) = λΦ(x) + ψΦ(y)—the map preserves linear combinations. This is the property that makes the map "structure-preserving."

Common examples include rotations (which preserve lengths and angles), projections (which collapse one dimension), and scaling (which stretches or compresses). Importantly, a linear map must send the zero vector to the zero vector: Φ(0) = Φ(0 · x) = 0 · Φ(x) = 0. This is why translation (adding a constant vector) is not linear—it shifts the origin. Affine maps combine a linear map with a translation: f(x) = Ax + b.

Every linear map is a matrix (and vice versa)

This is the central theorem of the section. Given a linear map Φ: V → W and chosen bases B = {b1, ..., bn} for V and C = {c1, ..., cm} for W, there is a unique m × n matrix AΦ such that the coordinate vector of Φ(x) with respect to C equals AΦ times the coordinate vector of x with respect to B. The j-th column of AΦ is the coordinate representation of Φ(bj) in basis C. Conversely, every m × n matrix defines a linear map from Rn to Rm.

This correspondence means that studying linear maps and studying matrices are the same thing (once bases are fixed). Different basis choices yield different matrices for the same map. Change of basis transforms the matrix representation: if B and B' are two bases for V related by a change-of-basis matrix S (so that the coordinates transform as x' = S-1x), then the matrix of Φ in the new basis is A' = T-1AS, where T is the change-of-basis matrix for W.

Injective, surjective, bijective

A linear map Φ: V → W is injective (one-to-one) if Φ(x) = Φ(y) implies x = y. Equivalently, ker(Φ) = {0}: the only vector mapping to zero is the zero vector itself. This means no information is lost. Surjective (onto) means Im(Φ) = W: every vector in the target space is reached. Bijective means both: the map is an isomorphism between V and W, meaning the two spaces are structurally identical. A bijective linear map has a linear inverse Φ-1: W → V, corresponding to the matrix inverse A-1.

In the matrix language: injective corresponds to rk(A) = n (full column rank, no non-trivial null space); surjective corresponds to rk(A) = m (full row rank, columns span the target space); bijective requires m = n and rk(A) = n (square and invertible). Neural network layers are typically neither injective nor surjective: they compress some dimensions (losing information) while not reaching all possible output vectors.

Key Insight

Once you fix bases for the source and target spaces, every linear map corresponds to exactly one matrix, and every matrix corresponds to exactly one linear map. Different basis choices give different matrices for the same underlying map. The map is the invariant; the matrix is its coordinate-dependent representation.

Section 08 — Details

Image, Kernel & Connections

For a linear map Φ: V → W (represented by matrix A), two subspaces capture everything about its behavior. The image (or range) Im(Φ) = {Φ(x) : x ∈ V} is the set of all outputs—the "reachable" part of W. For a matrix, this is the column space: the span of the columns of A. The kernel (or null space) ker(Φ) = {x ∈ V : Φ(x) = 0} is the set of all inputs that map to zero—the "invisible" part of V. These two subspaces are connected by the rank-nullity theorem: dim(Im Φ) + dim(ker Φ) = dim(V), or equivalently, rk(A) + nul(A) = n.

The rank-nullity theorem is a conservation law: the dimension of the domain splits into two pieces. The rank tells you how many independent dimensions survive the transformation. The nullity tells you how many dimensions are crushed to zero. If a 10-dimensional input is mapped by a rank-7 matrix, then 7 dimensions of information survive and 3 are lost. This is the mathematical essence of dimensionality reduction.

Strang's four fundamental subspaces

Gilbert Strang emphasizes that every m × n matrix A generates four subspaces, arranged in two orthogonal pairs. In Rn: the row space (dimension r = rank) and the null space (dimension n - r) are orthogonal complements—every vector in Rn splits uniquely into a row-space component and a null-space component. In Rm: the column space (dimension r) and the left null space (dimension m - r) are orthogonal complements. The matrix A maps the row space bijectively onto the column space, while collapsing the null space to zero. This "big picture" is one of the most clarifying frameworks in all of linear algebra.

Change of basis

The same linear map looks different depending on which bases you use. If you switch from basis B to basis B' in V using a change-of-basis matrix S (so x' = S-1x), the transformation matrix changes from A to S-1AS (for an endomorphism, where V = W). The eigendecomposition A = PDP-1 is a change of basis to the eigenvector basis, where the map becomes diagonal—just scaling along each eigenvector direction. This is the geometric insight behind diagonalization: find the right coordinate system, and the transformation becomes as simple as possible.

The rank-nullity theorem is a conservation law for dimensions. When a matrix maps Rn to Rm, it perfectly preserves r dimensions (the row space maps bijectively to the column space) and destroys n - r dimensions (the null space collapses to zero). No dimensions appear from nowhere; none vanish without a trace in the null space.
