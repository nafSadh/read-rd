§2.1

What is a Vector?

MML opens with a question most courses skip: what counts as a vector? The answer is anything you can add together and scale. This abstract view makes the same math work across geometry, polynomials, audio signals, and numerical arrays.

Definition (informal)
A vector is an element of a vector space.
Four concrete examples: geometric arrows, polynomials, audio signals, tuples in $\mathbb{R}^n$.
The two operations
$\mathbf{x} + \mathbf{y}$   and   $\lambda \mathbf{x}$
Addition and scalar multiplication — if these are defined and well-behaved, you have a vector space.
Key insight
The power of linear algebra is that the same theorems apply to all these different objects. Proving something for abstract vectors proves it for polynomials, signals, matrices — everything at once.
§2.2

Linear Systems (Ax = b)

The motivating problem of the entire chapter. Given a matrix $A$ and vector $b$, find all $\mathbf{x}$ such that $A\mathbf{x} = \mathbf{b}$. There are exactly three possible outcomes — no fourth option exists.

Three outcomes for Ax = b
Compact notation
$A\mathbf{x} = \mathbf{b}$
$A \in \mathbb{R}^{m \times n}$, $\mathbf{x} \in \mathbb{R}^n$, $\mathbf{b} \in \mathbb{R}^m$. Each row is one equation, each column of $A$ is a variable's coefficients.
Column view
$\mathbf{b} = x_1 \mathbf{a}_1 + \cdots + x_n \mathbf{a}_n$
Solving $A\mathbf{x} = \mathbf{b}$ asks: "is $\mathbf{b}$ a linear combination of $A$'s columns?" This connects directly to the column space.
§2.3

Matrices & Operations

Matrices are three things simultaneously: compact notation for systems of equations, collections of column vectors, and representations of linear transformations. The third view is the most powerful.

Matrix multiplication
$(AB)_{ij} = \sum_k A_{ik}B_{kj}$
Row $i$ of $A$ dotted with column $j$ of $B$. Dimensions: $(n \times k)(k \times m) = (n \times m)$. Neighboring dimensions must match.
Inverse
$AA^{-1} = I = A^{-1}A$
Only exists for square matrices with $\det(A) \neq 0$. Key trap: $(AB)^{-1} = B^{-1}A^{-1}$ (reversed order!).
Transpose
$(A^\top)_{ij} = A_{ji}$
Flip rows and columns. $(AB)^\top = B^\top A^\top$ (also reversed!). Symmetric matrix: $A = A^\top$.
Non-commutativity
$AB \neq BA$ in general
Rotating then shearing $\neq$ shearing then rotating. But $(AB)C = A(BC)$ — associativity always holds.
Common traps
$(A+B)^{-1} \neq A^{-1} + B^{-1}$  ·  $(AB)^{-1} = B^{-1}A^{-1}$  ·  $(AB)^\top = B^\top A^\top$  ·  Matrix multiply is NOT elementwise.
2×2 inverse formula & Hadamard product

2×2 shortcut: For $A = \begin{pmatrix}a & b \\ c & d\end{pmatrix}$, $$A^{-1} = \frac{1}{ad-bc}\begin{pmatrix}d & -b \\ -c & a\end{pmatrix}$$ Swap the diagonal, negate the off-diagonal, divide by the determinant.

Hadamard (element-wise) product: $(A \odot B)_{ij} = A_{ij}B_{ij}$. Common in code (NumPy's *), but not standard mathematical matrix multiplication. These two are completely different operations.

§2.4

Gaussian Elimination

The workhorse algorithm. Three elementary row operations preserve the solution set while transforming the matrix into a triangular "staircase" form (REF), from which solutions can be read off.

3 Elementary Operations
Swap · Scale · Add rows
These preserve the solution set. The goal is Row Echelon Form (REF): leading 1s in a staircase pattern, zeros below pivots.
Solution structure
$\mathbf{x} = \mathbf{x}_{\text{particular}} + \mathbf{x}_{\text{null}}$
General solution = one particular solution + the entire null space. Pivot columns = basic variables; non-pivot = free variables.
Computing A⁻¹
$[A | I] \xrightarrow{\text{row ops}} [I | A^{-1}]$
Augment with the identity, row-reduce until $A$ becomes $I$. What's on the right is $A^{-1}$.
The minus-1 trick for finding the null space

After reducing to RREF, extend the matrix to $n \times n$ by inserting rows with $-1$ on the diagonal wherever a pivot is missing. The columns corresponding to those $-1$ entries give you the null space basis directly — no extra solving required.

Example: if a $3 \times 4$ RREF has pivots in columns 1 and 3, extend to $4 \times 4$ by adding a row with $-1$ in column 2 and another with $-1$ in column 4. Columns 2 and 4 of the extended matrix span the null space.

§2.5

Vector Spaces & Subspaces

The formal framework. A vector space is a set $V$ with addition and scalar multiplication satisfying eight axioms. A subspace is a subset that is itself a vector space — and there's a simple three-part test.

Subspace test
$U \subseteq V$ is a subspace if...
(1) $\mathbf{0} \in U$  ·  (2) $\mathbf{x}, \mathbf{y} \in U \Rightarrow \mathbf{x}+\mathbf{y} \in U$  ·  (3) $\mathbf{x} \in U, \lambda \in \mathbb{R} \Rightarrow \lambda\mathbf{x} \in U$
Key examples
$\ker(A) = \{\mathbf{x} : A\mathbf{x} = \mathbf{0}\}$
The null space is always a subspace. The solution set of $A\mathbf{x} = \mathbf{b}$ (with $\mathbf{b} \neq \mathbf{0}$) is NOT a subspace (no zero vector).
Group → Vector Space hierarchy
Group: $(\mathcal{G}, \otimes)$ with closure, associativity, identity, inverse. Abelian = commutative. Vector space = two groups sharing a set, linked by distributivity. $GL(n, \mathbb{R})$ = all invertible $n \times n$ matrices under multiplication — a group but NOT a vector space.
§2.6

Linear Independence

A set of vectors is linearly independent if none can be written as a combination of the others — no vector is "redundant information." This is the key criterion for a basis.

Definition
$\lambda_1\mathbf{v}_1 + \cdots + \lambda_k\mathbf{v}_k = \mathbf{0}$
Linearly independent if the ONLY solution is $\lambda_1 = \cdots = \lambda_k = 0$. If a non-trivial solution exists, the vectors are dependent.
How to test
Write as columns, row-reduce
Non-pivot columns = dependent vectors. In $\mathbb{R}^n$, at most $n$ vectors can be linearly independent.
Geographic intuition (MML's example)

MML uses travel directions: the vector from Nairobi to Kampala and the vector from Kampala to Kigali are linearly independent — knowing one tells you nothing about the other. But the direct Nairobi→Kigali vector is dependent: it's the sum of the first two, hence "redundant."

In ML: if two features in your dataset are perfectly correlated, one is redundant — the data lives on a lower-dimensional subspace. PCA (Ch 10) formalizes this.

§2.7

Basis & Rank

A basis is the minimal set of vectors that spans a space — equivalently, the maximal linearly independent set. Every vector in the space has a unique representation in terms of a given basis. The rank of a matrix is the dimension of its column space.

Basis properties
Minimal spanning = maximal independent
Dimension = number of basis vectors. Not the size of the vectors! $\text{span}\{[0,1]^\top\}$ is 1D even though vectors live in $\mathbb{R}^2$.
Rank
$\text{rk}(A) = \text{rk}(A^\top)$
Row rank = column rank (always). Full rank: $\text{rk}(A) = \min(m,n)$ — all columns (or rows) are linearly independent.
Standard basis of $\mathbb{R}^3$
$\mathbf{e}_1, \mathbf{e}_2, \mathbf{e}_3$
$\mathbf{e}_1=[1,0,0]^\top$, etc. Any other basis works — the standard basis is just the most convenient default.
§2.8

Linear Mappings

A linear mapping $\Phi: V \to W$ preserves addition and scaling. Every matrix is a linear mapping; every linear mapping between finite-dimensional spaces is a matrix. The rank-nullity theorem is one of the most beautiful results in linear algebra.

Linearity condition
$\Phi(\lambda\mathbf{x} + \psi\mathbf{y}) = \lambda\Phi(\mathbf{x}) + \psi\Phi(\mathbf{y})$
3B1B's visual test: if gridlines stay parallel and evenly spaced after transformation, it's linear.
Image (range)
$\text{im}(\Phi) = \{\Phi(\mathbf{x}) : \mathbf{x} \in V\}$
All outputs the mapping can produce. Equals the column space of the transformation matrix.
Kernel (null space)
$\ker(\Phi) = \{\mathbf{x} \in V : \Phi(\mathbf{x}) = \mathbf{0}\}$
All inputs that get "squished to zero." If $\ker(\Phi) \neq \{\mathbf{0}\}$, the mapping is not injective — information is lost.
Rank-Nullity Theorem
$\text{rk}(A) + \dim(\ker(A)) = n$
Rank tells you the dimension of the output; nullity tells you what's lost. They always sum to the input dimension $n$.
Linear mapping: image and kernel

ker(Φ) maps entirely to 0. The rest of $V$ maps onto im(Φ). Rank-nullity: dim(ker) + dim(im) = dim(V).

Rank-nullity intuition
If $A$ is $3 \times 3$ with rank 2, it squishes 3D space onto a 2D plane. The lost dimension is the 1D null space — the direction that gets flattened. rank + nullity = 3 always.
Taxonomy of linear mappings
  • Injective (one-to-one): different inputs → different outputs. Equivalent to $\ker(\Phi) = \{\mathbf{0}\}$.
  • Surjective (onto): every output is reachable. Equivalent to $\text{im}(\Phi) = W$.
  • Bijective: both injective and surjective → invertible.
  • Isomorphism: bijective linear map. $V \cong W$ iff they have the same dimension.
  • Endomorphism: $\Phi: V \to V$ (same space in and out).
  • Automorphism: bijective endomorphism. Invertible $n \times n$ matrices are automorphisms of $\mathbb{R}^n$.

Transformation matrix: columns of $A_\Phi$ are the images of the basis vectors: $A_\Phi = [\Phi(\mathbf{b}_1) \mid \cdots \mid \Phi(\mathbf{b}_n)]$.

§2.9

Change of Basis & Affine Spaces

The same linear mapping looks different in different bases. Finding the basis that makes a matrix simple (diagonal) is what eigendecomposition achieves in Ch 4. Affine spaces are vector subspaces shifted away from the origin — they appear everywhere in regression and SVMs.

Basis change formula
$\tilde{A}_\Phi = T^{-1} A_\Phi S$
$S$ = basis change for $V$, $T$ = basis change for $W$. The mapping itself doesn't change — only its representation does.
Affine subspace
$L = \mathbf{x}_0 + U$
A subspace $U$ shifted by offset $\mathbf{x}_0$. Solution set of $A\mathbf{x} = \mathbf{b}$ (with $\mathbf{b} \neq 0$) is an affine subspace. Hyperplanes are affine subspaces of codimension 1.
Why this matters
Eigendecomposition (Ch 4) finds the basis where $A$ is diagonal — the "right" coordinate system for the matrix. This is the key idea behind PCA, Markov chains, and spectral methods.
