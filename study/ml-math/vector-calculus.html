Vector Calculus

Differentiation as linear approximation. Partial derivatives, Jacobians, matrix gradients, and backpropagation — all unified by the idea that a derivative is the best linear map approximating a function locally.

§5.1

Differentiation of Univariate Functions

A derivative answers: what is the best linear approximation to $f$ near this point? The limit definition captures slope as the infinitesimal rise-over-run, but the geometric meaning is a tangent line — the unique linear function that agrees with $f$ at $x$ to first order. Everything in this chapter generalizes this one idea.

Limit definition
$f'(x) = \lim_{h \to 0} \dfrac{f(x+h) - f(x)}{h}$
Slope of the tangent line at $x$. If $f'(x) > 0$, $f$ is increasing; $f'(x) < 0$, decreasing.
Chain rule (univariate)
$(g \circ f)'(x) = g'(f(x)) \cdot f'(x)$
The most important rule. In the multivariate setting it becomes a product of Jacobian matrices — the engine of backpropagation.
Product rule
$(fg)' = f'g + fg'$
Quotient rule follows: $(f/g)' = (f'g - fg')/g^2$.
Standard derivatives to know
Function $f(x)$Derivative $f'(x)$Notes
$x^n$$nx^{n-1}$Power rule; works for any $n \in \mathbb{R}$
$\exp(x)$$\exp(x)$Only function equal to its own derivative
$\ln(x)$$1/x$Appears everywhere in log-likelihoods
$\sin(x)$$\cos(x)$Rotate 90°: sin→cos→−sin→−cos→sin
$\cos(x)$$-\sin(x)$

The Taylor series (§5.8) shows why these derivatives have these forms — each derivative is the coefficient of the linear term in the local polynomial expansion.

The thread connecting this chapter
Every concept ahead — partial derivatives, gradients, Jacobians, backprop — is the univariate derivative generalized to higher dimensions. The derivative is always asking the same question: what linear map best approximates this function here?
§5.2

Partial Differentiation and Gradients

For $f : \mathbb{R}^n \to \mathbb{R}$, differentiation must handle multiple inputs simultaneously. The partial derivative $\partial f / \partial x_i$ isolates one input at a time. The gradient $\nabla_{\mathbf{x}} f$ collects all partials into a single object — and in MML, that object is a row vector.

Partial derivative
$\dfrac{\partial f}{\partial x_i} = \lim_{h \to 0} \dfrac{f(\mathbf{x} + h\mathbf{e}_i) - f(\mathbf{x})}{h}$
Hold all other inputs fixed; differentiate only $x_i$. $\mathbf{e}_i$ is the $i$-th standard basis vector.
Gradient — MML convention
$\nabla_{\mathbf{x}} f = \left[\frac{\partial f}{\partial x_1},\; \frac{\partial f}{\partial x_2},\; \ldots,\; \frac{\partial f}{\partial x_n}\right] \in \mathbb{R}^{1 \times n}$
Row vector! MML defines the gradient as a row vector. Many texts use a column vector — the convention matters when composing Jacobians.
Schwarz's theorem
$\dfrac{\partial^2 f}{\partial x_i \partial x_j} = \dfrac{\partial^2 f}{\partial x_j \partial x_i}$
Mixed partials commute when $f$ is twice continuously differentiable. Guarantees the Hessian (§5.7) is symmetric.
Gradient of squared norm
$\nabla_{\mathbf{x}} \|\mathbf{x}\|^2 = 2\mathbf{x}^\top$
The prototype calculation: $\|\mathbf{x}\|^2 = \sum_i x_i^2$, so $\partial/\partial x_i = 2x_i$. Result is a row vector.
Direction of steepest ascent
The gradient $\nabla f$ points in the direction of steepest ascent at a point. Its magnitude $\|\nabla f\|$ gives the rate of increase in that direction. Gradient descent moves opposite to the gradient — this is the entire algorithm: $\mathbf{x} \leftarrow \mathbf{x} - \eta \nabla f(\mathbf{x})^\top$.
Multivariate differentiation rules

The sum and product rules carry over directly:

  • Sum: $\nabla(f + g) = \nabla f + \nabla g$
  • Product: $\nabla(fg) = g\,\nabla f + f\,\nabla g$
  • Chain rule: For $h(\mathbf{x}) = f(g(\mathbf{x}))$, use Jacobians — see §5.3.

Example — for a linear function $f(\mathbf{x}) = \mathbf{a}^\top \mathbf{x}$:

$$\nabla_{\mathbf{x}} (\mathbf{a}^\top \mathbf{x}) = \mathbf{a}^\top \quad (\text{a row vector})$$
§5.3

Gradients of Vector-Valued Functions (Jacobians)

When $f : \mathbb{R}^n \to \mathbb{R}^m$ outputs a vector, the derivative is no longer a single row vector. It becomes the Jacobian matrix $J \in \mathbb{R}^{m \times n}$ — the matrix that best linearly approximates $f$ near a point. Each row is the gradient of one output; each column shows how all outputs respond to one input.

Jacobian as a linear map between two spaces
Jacobian matrix
$J = \dfrac{\partial \mathbf{f}}{\partial \mathbf{x}} \in \mathbb{R}^{m \times n}, \quad J_{ij} = \dfrac{\partial f_i}{\partial x_j}$
$m$ rows (one per output), $n$ columns (one per input). Row $i$ = gradient of $f_i$. Column $j$ = sensitivity of all outputs to $x_j$.
Chain rule (vector form)
$\dfrac{\partial \mathbf{z}}{\partial \mathbf{x}} = \underbrace{\dfrac{\partial \mathbf{z}}{\partial \mathbf{y}}}_{p \times m} \underbrace{\dfrac{\partial \mathbf{y}}{\partial \mathbf{x}}}_{m \times n}$
Matrix multiplication of Jacobians. Shapes: result is $p \times n$. This is backpropagation, traversed right to left.
Linearization
$\mathbf{f}(\mathbf{x}_0 + \Delta\mathbf{x}) \approx \mathbf{f}(\mathbf{x}_0) + J(\mathbf{x}_0)\,\Delta\mathbf{x}$
The Jacobian is the linear map that locally approximates $\mathbf{f}$. First-order Taylor expansion for vector functions.
Dimension tracking is everything
If $\mathbf{y} \in \mathbb{R}^m$ and $\mathbf{x} \in \mathbb{R}^n$, then $\partial\mathbf{y}/\partial\mathbf{x} \in \mathbb{R}^{m \times n}$. Always check shapes before multiplying Jacobians — a mismatch means you have the wrong convention or a transposed term. In ML: if the loss is scalar and parameters are a vector of length $n$, the gradient is $(1 \times n)$.
Example: neural network layer Jacobian

For a layer $\mathbf{y} = \sigma(W\mathbf{x} + \mathbf{b})$ where $\sigma$ acts elementwise, $\mathbf{y} \in \mathbb{R}^m$, $\mathbf{x} \in \mathbb{R}^n$:

$$\frac{\partial \mathbf{y}}{\partial \mathbf{x}} = \underbrace{J_\sigma(W\mathbf{x}+\mathbf{b})}_{m \times m} \cdot \underbrace{W}_{m \times n} \in \mathbb{R}^{m \times n}$$

$J_\sigma$ is diagonal (elementwise activation): $J_\sigma = \operatorname{diag}(\sigma'(W\mathbf{x}+\mathbf{b}))$. This is why pointwise activations are computationally cheap to differentiate.

The gradient with respect to $W$ is needed for parameter updates: $\partial L/\partial W = (\partial L/\partial \mathbf{y}) \cdot \partial \mathbf{y}/\partial W$, which involves a Kronecker product structure — handled automatically by AD frameworks.

§5.4

Gradients of Matrices

When the input or output is a matrix, the gradient inherits the same shape. For $f : \mathbb{R}^{m \times n} \to \mathbb{R}$ (a scalar function of a matrix, like a loss w.r.t. weight matrix $W$), the gradient is also an $m \times n$ matrix — one entry per parameter.

Gradient w.r.t. matrix
$\left(\dfrac{\partial f}{\partial A}\right)_{ij} = \dfrac{\partial f}{\partial A_{ij}}$
Same shape as $A$. Differentiate $f$ w.r.t. each entry independently; pack the results back into a matrix.
Useful matrix identities
$\dfrac{\partial}{\partial X}[\mathbf{a}^\top X \mathbf{b}] = \mathbf{a}\mathbf{b}^\top$
Also: $\partial\,\mathrm{tr}(AXB) / \partial X = A^\top B^\top$. These appear when differentiating quadratic losses w.r.t. weight matrices.
Vectorization (vec operator)
$\operatorname{vec}(A) \in \mathbb{R}^{mn}$
Stack columns of $A$ into one long vector. Converts matrix calculus into standard vector Jacobians — used when notationally consistent bookkeeping is needed.
Why this matters for ML
Every weight matrix $W$ in a neural network is differentiated via $\partial L / \partial W$. The result is a matrix of the same shape — used directly in gradient descent. Understanding this gradient as "same shape as the parameter" is the foundation of parameter update rules: $W \leftarrow W - \eta\,(\partial L / \partial W)$.
§5.5

Useful Identities for Computing Gradients

A reference collection of frequently needed matrix-calculus results. Derivation strategy: write out element-by-element, take one partial, read off the pattern, pack into matrix form, check dimensions. The log-determinant identity is especially important for Gaussian models.

$\mathbf{x}^\top B \mathbf{x}$$\mathbf{x}^\top(B + B^\top)$ w.r.t. $\mathbf{x}$Bilinear form

Why is the log-det gradient A⁻ᵀ?

Jacobi's formula states $\mathrm{d}\det(A) = \det(A)\,\mathrm{tr}(A^{-1}\,\mathrm{d}A)$, so $\partial\det(A)/\partial A_{ij} = \det(A)\,(A^{-1})_{ji}$, giving $\partial\det(A)/\partial A = \det(A)\cdot A^{-\top}$.

Taking the log: $\partial\ln\det(A)/\partial A = (1/\det(A)) \cdot \det(A) \cdot A^{-\top} = A^{-\top}$.

In maximum-likelihood estimation for a Gaussian with covariance $\Sigma$, the log-likelihood contains $-\tfrac{1}{2}\ln\det(\Sigma)$. Differentiating w.r.t. $\Sigma$ gives $-\tfrac{1}{2}\Sigma^{-\top} = -\tfrac{1}{2}\Sigma^{-1}$ (symmetric $\Sigma$). This is the gradient that drives the MLE update for the covariance.

§5.6

Backpropagation and Automatic Differentiation ⭐

The most important section for ML practitioners. Backpropagation is not a special algorithm — it is the chain rule applied to Jacobians, traversed in reverse order through a computation graph. Two passes: forward (compute values), backward (accumulate gradients). This enables exact, efficient gradient computation through networks with millions of parameters.

Computation graph — forward values and backward gradients
Forward mode AD
Input → Output
  • Propagates $\partial y_j / \partial x_i$ forward
  • One sweep per input variable
  • Cost: $O(n)$ for $n$ inputs
  • Best when few inputs, many outputs
  • Primitive: jvp (Jacobian-vector product)
Reverse mode AD (Backprop)
Output → Input ⭐
  • Propagates $\partial L / \partial x_i$ backward
  • One sweep per output variable
  • Cost: $O(m)$ for $m$ outputs
  • Best when few outputs, many inputs
  • Primitive: vjp (vector-Jacobian product)
Why reverse mode wins for ML
Neural networks have one scalar loss ($m = 1$) and millions of parameters ($n \approx 10^6$–$10^9$). Forward mode would require $n$ separate sweeps — one per parameter. Reverse mode requires only one backward sweep regardless of $n$. This single asymmetry is why backpropagation is the foundation of all deep learning training.
VJP — the reverse-mode primitive
$\mathbf{v}^\top J_f(\mathbf{x}) \in \mathbb{R}^{1 \times n}$
Vector-Jacobian product: upstream gradient $\mathbf{v}^\top$ times local Jacobian $J_f$. Avoids materializing the full $m \times n$ matrix — each operation contributes only this compact product.
Reverse accumulation (backprop)
$\dfrac{\partial L}{\partial \mathbf{y}_i} = \dfrac{\partial L}{\partial \mathbf{y}_{i+1}} \cdot \dfrac{\partial \mathbf{y}_{i+1}}{\partial \mathbf{y}_i}$
Chain rule applied layer by layer, right to left. Each step multiplies the running gradient by a local Jacobian. This is all backprop is.
Example: one linear layer backward pass

For $\mathbf{y} = W\mathbf{x} + \mathbf{b}$ with loss $L = \tfrac{1}{2}\|\mathbf{y} - \mathbf{t}\|^2$:

Forward pass: compute $\mathbf{y}$, store $\mathbf{x}$ and $\mathbf{y}$.

Backward pass:

  • $\partial L / \partial \mathbf{y} = (\mathbf{y} - \mathbf{t})^\top \quad [1 \times m]$
  • $\partial L / \partial W = (\mathbf{y} - \mathbf{t})\mathbf{x}^\top \quad [m \times n]$ — used to update $W$
  • $\partial L / \partial \mathbf{b} = (\mathbf{y} - \mathbf{t}) \quad [m \times 1]$ — used to update $\mathbf{b}$
  • $\partial L / \partial \mathbf{x} = W^\top(\mathbf{y} - \mathbf{t}) \quad [n \times 1]$ — passed to previous layer

Each gradient has exactly the shape of the quantity it differentiates. Frameworks compute these automatically from the forward computation graph.

AD vs. other differentiation methods
MethodMechanismProblem
SymbolicAlgebraically differentiates expressionsExpression size explodes; impractical for large graphs
NumericalFinite differences: $[f(x+h)-f(x)]/h$Truncation error; $O(n)$ evaluations per gradient

Tape / Wengert list: modern AD systems record all operations during the forward pass (building a "tape"), then replay them in reverse during the backward pass. The user only writes forward code; the backward pass is generated automatically.

§5.7

Higher-Order Derivatives: Hessians

The gradient tells you slope. The Hessian tells you curvature — how the slope is changing. For $f : \mathbb{R}^n \to \mathbb{R}$, the Hessian is an $n \times n$ symmetric matrix whose eigenvalues describe the shape of the loss landscape around any point.

Hessian matrix
$H_{ij} = \dfrac{\partial^2 f}{\partial x_i \partial x_j}, \quad H \in \mathbb{R}^{n \times n}$
By Schwarz's theorem, $H$ is symmetric. Also written $\nabla^2 f$.
Definiteness and critical points
$H \succ 0 \Rightarrow$ local min   $H \prec 0 \Rightarrow$ local max
At a critical point ($\nabla f = 0$): PD Hessian → minimum; ND → maximum; indefinite (mixed eigenvalues) → saddle point.
Newton's method (2nd order)
$\mathbf{x}_{t+1} = \mathbf{x}_t - H^{-1}\nabla f(\mathbf{x}_t)$
Quadratic convergence near the optimum vs. linear for gradient descent. Cost: $O(n^3)$ to invert $H$ — prohibitive for $n \sim 10^6$. Quasi-Newton (L-BFGS) approximates $H^{-1}$ cheaply.
The loss landscape and conditioning
The Hessian's eigenvalues describe curvature in each direction. A large ratio $\lambda_{\max}(H) / \lambda_{\min}(H)$ (ill-conditioned $H$) means the loss surface is steep in some directions and very flat in others. Gradient descent oscillates in steep directions and crawls in flat ones — this is part of why learning rate tuning and adaptive optimizers (Adam, RMSProp) matter: they implicitly normalize for Hessian conditioning.
§5.8

Linearization and Multivariate Taylor Series

The Taylor expansion gives a polynomial approximation to $f$ around any point $\mathbf{x}_0$. The first-order term is exactly the gradient (linear approximation); the second-order term adds the Hessian (quadratic approximation). Optimization algorithms differ in how many terms they use.

Univariate Taylor series
$f(x) = \sum_{k=0}^{K} \dfrac{f^{(k)}(x_0)}{k!}(x-x_0)^k + \text{rem.}$
$k=0$: constant. $k=1$: linear. $k=2$: quadratic. Derivatives are the coefficients.
Multivariate Taylor (2nd order)
$f(\mathbf{x}) \approx f(\mathbf{x}_0) + \nabla f(\mathbf{x}_0)(\mathbf{x}-\mathbf{x}_0) + \tfrac{1}{2}(\mathbf{x}-\mathbf{x}_0)^\top H(\mathbf{x}_0)(\mathbf{x}-\mathbf{x}_0)$
$\nabla f$ is a row vector; $H$ is $n \times n$ symmetric; $(\mathbf{x}-\mathbf{x}_0)$ is a column vector. Shapes work out to a scalar.
OrderApproximationUsed in
0th$f(\mathbf{x}_0)$ — constantBaseline; no gradient info
1st$f(\mathbf{x}_0) + \nabla f \cdot \Delta\mathbf{x}$ — linearGradient descent, linearization
2nd$+\,\tfrac{1}{2}\Delta\mathbf{x}^\top H\,\Delta\mathbf{x}$ — quadraticNewton's method, Laplace approximation
HigherPolynomial in $\Delta\mathbf{x}$Rarely used in practice
Linearization = Jacobian
The first-order Taylor expansion $\mathbf{f}(\mathbf{x}_0 + \Delta\mathbf{x}) \approx \mathbf{f}(\mathbf{x}_0) + J(\mathbf{x}_0)\,\Delta\mathbf{x}$ makes precise the definition of the derivative as a linear map. The Jacobian is the linearization operator. This unifies §5.1–5.3 into one statement.
Why optimizers differ: 1st vs 2nd order

Gradient descent uses only the 1st-order term: step size $\eta$ is chosen manually (the learning rate). It cannot "see" curvature — it treats all directions equally, which causes oscillation in ill-conditioned landscapes.

Newton's method minimizes the 2nd-order Taylor approximation exactly, yielding the step $-H^{-1}\nabla f$. This accounts for curvature: it takes large steps in flat directions and small steps in curved ones. Quadratic convergence near the optimum.

Momentum / Adam: approximate the second-order correction cheaply using running averages of gradients and squared gradients. Adam's denominator $\sqrt{\hat{v}_t} + \epsilon$ implicitly scales by $\approx \sqrt{H_{ii}}$ for diagonal entries — a diagonal Hessian approximation.

L-BFGS: a quasi-Newton method that builds a low-rank approximation to $H^{-1}$ from gradient history, without ever forming the full matrix. Used when exact Hessians are too expensive.
