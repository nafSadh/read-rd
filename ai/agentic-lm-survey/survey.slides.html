§3.1 · POMDP 5

The Mathematical Distinction

RLHF is a degenerate single-step MDP: one prompt, one response, one scalar reward. Horizon T=1.

Agentic RL is a temporally extended POMDP: horizon T>1, stochastic transitions via tool use, step-wise rewards, partial observability.

POMDP Tuple ⟨S, O, A, P, R, T, γ⟩

Action space decomposes: A_text (reasoning — deterministic) ∪ A_action (tool calls — stochastic).[2]Zhang et al. "Landscape of Agentic RL for LLMs." TMLR Jan 2026. arXiv:2509.02547

Why this matters

The RL training loop must handle both reasoning and tool-use within a single policy — fundamentally different from chat alignment.[3]Cheng et al. "Agent-R1." Nov 2025. arXiv:2511.14460

Zhang et al. (25 authors), "Landscape of Agentic RL," TMLR Jan 2026. 500+ works synthesized.

§3.2 · Scaling 6

Scaling Laws and Efficiency

Power-law scaling for RL post-training: 63 models (0.5B–72B) show predictive fit. Learning efficiency saturates with model size. Data reuse is surprisingly effective.[4]Tan et al. "Scaling Behaviors of LLM RL Post-Training." Dec 2025. arXiv:2509.25300

Scaling Law log L(N,X) = −k(N) · log X + E(N)

Agent-Omit quantified the waste: 45.1% thoughts, 52.2% observations, 2.7% actions. Trains agents to skip redundant reasoning — 8B matches frontier agents.[5]Ning et al. "Agent-Omit." Feb 2026. arXiv:2602.04284

ACT (Mar 2026): Training agents to judge action quality improves MATH-500 and GPQA without reasoning-specific data. Agentic RL → general reasoning transfer.[7]Liu et al. "Agentic Critical Training." UMD, Mar 2026. arXiv:2603.08706

Implication

4B models with proper RL recipes can match 32B baselines. Efficiency is becoming a first-class training objective.[6]Yu et al. "Demystifying RL in Agentic Reasoning." Oct 2025. arXiv:2510.11701

§4.1 · Benchmarks 8

The Benchmark Reality Check

SWE-Bench Pro: 1,865 enterprise problems, 41 repos. Best: <45%. Without augmentations: GPT-5 drops 25.9% → 8.4%.[8]Deng et al. "SWE-Bench Pro." Scale AI, Nov 2025. arXiv:2509.16941

SWE-EVO: Long-horizon evolution. GPT-5 achieves 65% → 21% — a 44pp drop.[9]Thai et al. "SWE-EVO." FPT/Melbourne, Jan 2026. arXiv:2512.18470

Saving SWE-Bench: Realistic user queries reveal 20–50% overestimation.[11]Garg et al. "Saving SWE-Bench." Microsoft, CAIN '26. arXiv:2510.08996

UTBoost: 345 erroneous patches. 40.9% of leaderboard affected.[12]Yu et al. "UTBoost." CUHK-SZ/UIUC, Jun 2025. arXiv:2506.09289

SWE-bench Live: Continuous updates show performance gaps vs static benchmarks — contamination.[10]Zhang et al. "SWE-bench Goes Live!" Microsoft, Jun 2025. arXiv:2505.23419

Bottom line

Headline numbers are inflated. Real capabilities are considerably below reported scores.

§4.2 · Self-Improvement 9

Self-Improvement Beats Hand-Crafting

Live-SWE-agent (UIUC): Starts from minimal bash scaffold, creates tools on-the-fly. 77.4% SWE-bench Verified. Zero offline cost.[13]Xia et al. "Live-SWE-agent." UIUC, Nov 2025. arXiv:2511.13646

Self-Play SWE-RL (Meta FAIR): Bug-injector + solver, no human data. +10.4 over pre-RL baseline, beats human-data RL.[14]Wei et al. "Self-Play SWE-RL." Meta FAIR/UIUC/CMU, Dec 2025. arXiv:2512.18552

RepoNavigator (PKU): One "jump" tool + RL. 7B beats 14B multi-tool baselines.[15]Zhang et al. "One Tool Is Enough." PKU/Baidu, Jan 2026. arXiv:2512.20957

The pattern

Self-generated tools, self-generated training data, and minimal tool design with RL each independently outperform hand-crafted alternatives.

Minimum model capability threshold required — below GPT-5-Mini, self-improvement degrades.[13]Xia et al. "Live-SWE-agent." UIUC, Nov 2025. arXiv:2511.13646

§5.1 · Protocols 11

The Agentic Protocol Stack

MCP (Anthropic): JSON-RPC 2.0 for tool invocation. 97M+ monthly SDK downloads. "How agents talk to tools."[17]Ehtesham et al. "Agent Interoperability Protocols." May 2025. arXiv:2505.02279

A2A (Google): Agent Cards for capability discovery. Peer-to-peer task delegation. "How agents talk to each other."

AAIF (Dec 2025): Six competitors — OpenAI, Anthropic, Google, Microsoft, AWS, Block — co-governing standards under Linux Foundation.

Multi-agent orchestration: 348 trials showed 100% actionable vs 1.7% single-agent, with zero quality variance.[16]Drammeh. "Multi-Agent LLM Orchestration." Jan 2026. arXiv:2511.15755

The pattern

MCP for the resource layer, A2A for the network layer. They compose like TCP/IP, not compete.

§6 · Security 13

Four Numbers That Define the Problem

23–41%: MCP amplifies attack success rates compared to equivalent non-MCP integrations. Architectural, not implementation.[19]Maloyan & Namiot. "Breaking the Protocol." Jan 2026. arXiv:2601.17549

78%+: All tested defenses bypassed under adaptive attacks (PromptGuard 91%, Protect AI 93%).[20]Maloyan & Namiot. "Prompt Injection SoK." Jan 2026. arXiv:2601.17548

57%: Of MCP tool call chains leak sensitive data by default — not adversarial, just systemic over-exposure.[21]Lin et al. "AgentRaft." Sun Yat-sen/UCF, Mar 2026. arXiv:2603.07557

26.1%: Of community-contributed agent skills contain security vulnerabilities (42,447 analyzed).[22]Xu & Yan. "Agent Skills Survey." Zhejiang, Feb 2026. arXiv:2602.12430

The thesis

Security is architectural, not patchable. Prompt injection needs architectural mitigation like SQL injection got parameterized queries.

Skills as Portable Capability

Formal definition: S = (C, π, T, R) — applicability condition, executable policy, termination condition, reusable interface.[23]Jiang et al. "SoK: Agentic Skills." UTS/CSIRO, Feb 2026. arXiv:2602.20867

Progressive disclosure: L1 metadata (~30 tokens, always loaded) → L2 instructions (200–2K, on trigger) → L3 resources (on demand).[22]Xu & Yan. "Agent Skills Survey." Zhejiang, Feb 2026. arXiv:2602.12430

1,300+ skills across 8+ tools. Contributions from Anthropic, Google DeepMind, OpenAI, Microsoft, Vercel.[27]Anthropic. "2026 Agentic Coding Trends Report." Jan 2026

Skills + MCP are complementary: skills = "what to do," MCP = "how to connect."

Five configuration layers

Model → Project (CLAUDE.md) → Skills (SKILL.md) → Tools (MCP) → Agent-to-Agent (A2A). Trust decreases down the stack.

VII
§8 · Findings 17

The Five Findings

1. Scaffold→weights migration is real. RL internalizes planning, tool use, memory, reflection into model weights. ACT: agentic RL → general reasoning transfer.

2. Self-improvement outperforms hand-crafting. Live-SWE 77.4%, SSR +10.4, RepoNavigator 7B>14B. Threshold: minimum model capability required.

3. Benchmarks overestimate, production underdelivers. SWE-EVO 65%→21%. UTBoost: 40.9% erroneous patches. The 60/20 paradox.

4. Interaction became infrastructure. MCP 97M+ downloads. AAIF governance. Flow engineering > full autonomy.

5. Security is architectural, not patchable. MCP amplifies 23–41%. All defenses bypassed 78%+. 57% tool chains leak. 26.1% skills contain vulns.

§8 · Agenda 18

Research Agenda

Close the long-horizon gap: SWE-EVO's 65%→21% is the clearest open problem. Multi-file, multi-session reasoning across hours of work.

Agentic scaling laws: Existing laws are for pure reasoning. How does tool-augmented RL shift the compute-optimal frontier?

Protocol-level security: Incorporate AttestMCP/SMCP into the MCP spec itself. Solve the cross-protocol identity gap.

Data Over-Exposure mitigation: Protocol-layer data minimization. Tools should return only the data requested, not full schemas.

Skill governance at scale: Same rigor as npm/pip. Signed manifests, behavioral verification, trust-tiered deployment.

Bridge benchmarks to production: Measure sustained, collaborative, multi-session coding — not isolated issue resolution.
