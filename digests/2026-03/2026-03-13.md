## Research Highlights

### When Agents Fail to Act: A 12-Category Error Taxonomy for Tool-Use in Multi-Agent Systems
**Category:** Research Paper | **Source:** arXiv

Published as part of the VoltAgent research collection, this paper presents the first systematic diagnostic framework for production agent reliability, cataloging 12 distinct failure categories observed across real deployments. The taxonomy covers planning errors, tool selection failures, hallucinated API calls, retry loops, context overflow, inter-agent handoff breakdowns, and more — each with representative examples, root-cause patterns, and proposed mitigations. Unlike benchmark-focused work, the failure modes documented here are the ones that surface in production but rarely appear in standard evaluations, making this immediately applicable for teams shipping agent systems.

[Read more](https://arxiv.org/abs/2603.09015)

### AI Model Predicts Cancer Metastasis Probability with Better-Than-Expert Accuracy
**Category:** Research Paper | **Source:** ScienceDaily / arXiv (preprint)

A new machine learning model trained on tumor genomics data achieves better-than-expert performance at predicting the probability that a given cancer will metastasize, according to coverage by ScienceDaily ahead of the formal arXiv release. The model was evaluated against oncologist predictions on held-out patient cohorts, with statistically significant improvements in both sensitivity and specificity. If results hold through peer review, it represents a genuine clinical inflection point — routine biopsy data alone, processed by a standard ML pipeline, could inform treatment planning decisions that currently require expensive specialist interpretation.

## Agent & Tool Use

### The 12-Category Agent Error Taxonomy: A Reference Framework for Production Teams
**Category:** Research Paper | **Source:** arXiv

The "When Agents Fail to Act" taxonomy fills a genuine gap in the agent engineering literature. Benchmark evaluations measure task success rates but rarely expose the failure pathways that matter most in deployment — what happens when a tool returns an unexpected schema, when an agent loops instead of escalating, or when context overflow silently degrades decision quality mid-task. Having a named, shared vocabulary for these failure modes enables teams to instrument their systems more precisely, write targeted regression tests, and communicate failure patterns across engineering and product functions without ambiguity.
