01

Code as Constructed Language

Programming languages are, by any reasonable definition, constructed languages. They are formal systems designed to express meaning in a structured notation. They have syntax, semantics, and pragmatics. They are learned, they form communities, and they evolve over time.[1]Wikipedia: Esoteric programming language →

Yet the conlang and programming language communities rarely recognize each other. Both are engaged in the same fundamental activity: designing formal systems for expressing meaning. The design challenges are remarkably similar: balancing expressiveness with learnability, handling ambiguity, building communities of users.

APL: Radical Notation

Kenneth Iverson’s APL (1962) is arguably the most conlang-like programming language ever created. It uses a unique character set that looks nothing like conventional mathematics or programming.[2]Esolang Wiki: APL as radical notation →

Iverson received the Turing Award in 1979. His lecture was titled “Notation as a Tool of Thought” — a direct engagement with the Sapir-Whorf hypothesis applied to programming. His argument: the notation you use shapes how you think about problems. APL’s array notation makes certain mathematical insights visible that procedural languages obscure.

Tension T5: Formal vs. Natural Notation

APL proves that notation shapes thought. Its radical departure from conventional syntax changed how programmers think about computation — the strongest evidence for linguistic relativity in the computational domain.

Brainfuck is the purest demonstration that a Turing-complete language can be almost impossibly minimal. It is a reductio ad absurdum of language design. But it is more than a joke — it is used to teach computational theory and is, arguably, art.[4]The Outline: Brainfuck — code designed to hurt →

Tension T6: Minimalism vs. Richness

Brainfuck (8 commands, Turing-complete) vs. Tolkien’s Quenya (thousands of words, full etymology). Both are valid expressions of language as art — at opposite extremes of the same creative impulse.

Dijkstra vs. Natural Language Programming

In 1978, Edsger Dijkstra wrote “On the foolishness of ‘natural language programming’” (EWD 667).[7]Dijkstra EWD 667: Formal symbolism is essential (1978) → His argument: formal symbolism is essential to precise thought. Greek mathematics stalled because it was verbal. Modern science emerged via formal notation.

Then came LLMs. In 2025–2026, people write code by describing what they want in English. Marc Brooker’s response to Dijkstra: LLMs have made natural language programming partially successful.[8]Brooker: On the success of natural language programming (2025) → But formal languages remain necessary for precision, verification, and debugging.

Dijkstra’s Position

Formalism is the point. Programming languages should be as unlike natural language as possible. Ambiguity is the enemy.

The LLM Response

Natural language programming works now — partially. But formal languages remain essential for precision. Both are right.

06

Programming Communities Mirror Conlang Communities

The structural parallels between conlang and programming language communities are striking:

Esperanto ↔ Python

Utility-focused, community-driven. Both survived through strong community identity rather than technical superiority alone.

Quenya ↔ Haskell

Mathematically beautiful, intellectually deep. Both prioritize elegance over accessibility.

Klingon ↔ JavaScript

Both grew wildly beyond their original purpose. Neither was designed to do what it now does.

Volapuk ↔ Governance Failures

Languages destroyed by founder control. The pattern recurs in both domains.

Brainfuck ↔ Brainfuck

It is literally both — a constructed language and a programming language. The purest overlap.

Lojban ↔ Prolog

Logical, designed to shape thinking. Both have small but fiercely dedicated communities.

07

Regex: The Worst Conlang Ever

Regular expressions deserve special mention as a constructed language that is universally used, widely criticized, syntactically dense to the point of being write-only, learned by memorizing arcane symbols, and essential despite its poor readability.

Regex is the Volapuk of programming: it works, but its users widely acknowledge its deficiencies. It evolved from theoretical computer science (regular languages in the Chomsky hierarchy) through Unix tools into a practical but notoriously opaque notation. Billions of lines of code depend on it, yet few practitioners would advocate its syntax as a model for new language design.

Agreement & Debate

Sources Agree

  • Programming languages are designed formal systems with syntax and semantics
  • APL demonstrates that notation genuinely shapes thought
  • Esolangs parallel the conlang tradition as creative language design
  • The natural language programming debate has been transformed by LLMs
  • Programming communities mirror conlang communities structurally

Sources Disagree

  • Whether programming languages are “really” languages in the linguistic sense
  • Whether Dijkstra’s argument has been refuted by LLMs
  • Whether esolangs are art, humor, pedagogy, or all three
  • Whether the conlang–programming parallel is deep or superficial

Cross-Cutting Finding #5

Programming languages are the most successful constructed languages ever created. By any measure — speakers, utility, economic impact — Python dwarfs Esperanto. Yet the two communities rarely talk. Both are engaged in the same fundamental activity: designing formal systems for expressing meaning.
